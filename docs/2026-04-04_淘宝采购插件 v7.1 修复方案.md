# 淘宝采购插件 v7.1 修复方案

**问题**: 单品采购数量 > 1 时卡顿  
**时间**: 2026-04-04  
**版本**: v7.1

---

## 🐛 问题分析

### 当前逻辑（v6.8）

```javascript
const setQuantity = async () => {
    if (targetQty <= 1) return;
    
    // 找到数量输入框
    const input = document.querySelector('.mui-amount-input');
    
    if (input) {
        // 清除原有值
        input.value = '';
        input.dispatchEvent(new Event('input', { bubbles: true }));
        
        // 设置新值
        await new Promise(r => setTimeout(r, 300));
        input.value = targetQty.toString();
        input.dispatchEvent(new Event('input', { bubbles: true }));
        input.dispatchEvent(new Event('change', { bubbles: true }));
        
        await new Promise(r => setTimeout(r, 1200));
    }
};
```

### 问题点

1. **清空操作可能触发验证** - 淘宝可能会在输入框为空时显示错误提示
2. **事件触发顺序** - 可能缺少 `focus` 和 `blur` 事件
3. **等待时间不足** - 淘宝可能需要更长时间验证价格/库存
4. **未处理输入框禁用状态** - 某些商品可能限制购买数量

---

## ✅ 修复方案

### 方案 A: 改进输入方式（推荐）

**核心思路**: 使用 `focus` → 清空 → 输入 → `blur` 的完整流程

```javascript
const setQuantity = async () => {
    if (targetQty <= 1) {
        console.log('[setQuantity] 数量为 1，跳过');
        return;
    }
    
    log(`🎯 设置数量：${targetQty}`);
    sub('查找数量输入框...');
    
    const quantitySelectors = [
        '.mui-amount-input', '#J_IptAmount', '.amount-input',
        '.quantity-inner input', '[name="quantity"]', '#quantity'
    ];
    
    let input = null;
    for (const s of quantitySelectors) {
        input = document.querySelector(s);
        if (input) break;
    }
    
    if (!input) {
        console.log('[setQuantity] 未找到数量输入框');
        sub('未找到数量输入框，使用默认值');
        return;
    }
    
    console.log('[setQuantity] 找到输入框:', input);
    sub('设置采购数量...');
    
    // 1. 聚焦输入框
    input.focus();
    await new Promise(r => setTimeout(r, 200));
    
    // 2. 全选原有内容
    input.select();
    await new Promise(r => setTimeout(r, 100));
    
    // 3. 直接设置新值（避免清空触发验证）
    const oldValue = input.value;
    input.value = targetQty.toString();
    
    // 4. 触发完整的事件链
    input.dispatchEvent(new Event('input', { bubbles: true }));
    input.dispatchEvent(new Event('change', { bubbles: true }));
    input.dispatchEvent(new Event('blur', { bubbles: true }));
    
    console.log('[setQuantity] 数量已设置:', targetQty);
    
    // 5. 等待淘宝验证（价格/库存）
    sub('等待页面更新...');
    await new Promise(r => setTimeout(r, 2000));
    
    // 6. 验证是否设置成功
    const actualValue = parseInt(input.value) || 1;
    if (actualValue !== targetQty) {
        console.warn('[setQuantity] 验证失败，实际值:', actualValue);
        sub('数量设置失败，使用默认值');
    } else {
        console.log('[setQuantity] 验证成功');
        sub('数量设置完成');
    }
};
```

### 方案 B: 使用键盘事件模拟

**核心思路**: 模拟真实用户键盘输入

```javascript
const setQuantityWithKeyboard = async (input, targetQty) => {
    input.focus();
    await new Promise(r => setTimeout(r, 200));
    
    // Ctrl+A 全选
    input.dispatchEvent(new KeyboardEvent('keydown', { key: 'a', ctrlKey: true }));
    await new Promise(r => setTimeout(r, 100));
    
    // 删除
    input.dispatchEvent(new KeyboardEvent('keydown', { key: 'Backspace' }));
    await new Promise(r => setTimeout(r, 100));
    
    // 逐位输入数字
    const digits = targetQty.toString().split('');
    for (const digit of digits) {
        input.dispatchEvent(new KeyboardEvent('keydown', { key: digit }));
        input.dispatchEvent(new KeyboardEvent('keypress', { key: digit }));
        input.value += digit;
        input.dispatchEvent(new Event('input', { bubbles: true }));
        await new Promise(r => setTimeout(r, 50));
    }
    
    input.dispatchEvent(new Event('change', { bubbles: true }));
    input.dispatchEvent(new KeyboardEvent('blur', { bubbles: true }));
    
    await new Promise(r => setTimeout(r, 1500));
};
```

**核心思路**: 如果设置失败，多次重试

const setQuantityWithRetry = async (maxRetries = 3) => {

// 设置数量（v7.1 修复版）
const setQuantity = async () => {
    if (targetQty <= 1) {
        console.log('[setQuantity] 数量为 1，跳过');
        return;
    }
    
    log(`🎯 设置数量：${targetQty}`);
    sub('查找数量输入框...');
    
    const quantitySelectors = [
        '.mui-amount-input', '#J_IptAmount', '.amount-input',

        console.log('[setQuantity] 未找到数量输入框');

        return;
    }
    
    console.log('[setQuantity] 找到输入框:', input);
    sub('设置采购数量...');
    
    try {
        // 1. 聚焦输入框
        input.focus();
        await new Promise(r => setTimeout(r, 200));
        
        // 2. 全选原有内容
        input.select();
        await new Promise(r => setTimeout(r, 100));
        
        // 3. 直接设置新值

        // 4. 触发完整事件链
        input.dispatchEvent(new Event('input', { bubbles: true }));
        input.dispatchEvent(new Event('change', { bubbles: true }));
        input.dispatchEvent(new Event('blur', { bubbles: true }));
        
        console.log('[setQuantity] 数量已设置:', targetQty);
        
        // 5. 等待淘宝验证
        sub('等待页面更新...');
        await new Promise(r => setTimeout(r, 2000));
        
        // 6. 验证设置结果
        const actualValue = parseInt(input.value) || 1;
        if (actualValue !== targetQty) {
            console.warn('[setQuantity] 验证失败，实际值:', actualValue);
            sub('数量设置失败，继续加购');
        } else {
            console.log('[setQuantity] 验证成功');
            sub('✓ 数量设置完成');
        }
        
    } catch (e) {
        console.error('[setQuantity] 异常:', e);
        sub('数量设置异常，继续加购');
    }
};

## 🎯 关键改进点

### v6.8 → v7.1

| 项目 | v6.8 | v7.1 | 改进 |
|------|------|------|------|
| **清空方式** | `value = ''` | `select()` 全选 | 避免触发空值验证 |
| **事件触发** | input + change | input + change + blur | 更完整的事件链 |
| **等待时间** | 1200ms | 2000ms | 给淘宝足够验证时间 |
| **验证步骤** | ❌ 无 | ✅ 验证实际值 | 确保设置成功 |
| **错误处理** | ❌ 无 | ✅ try-catch | 防止异常中断 |
