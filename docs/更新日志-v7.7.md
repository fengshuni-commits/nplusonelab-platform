## 📦 修复详情

### 问题 1: 整体产品订单价格不对

**原因**: 之前使用 `dataset.price`（零件包售价）  
**修复**: 根据订单类型动态计算价格

```javascript
async function onProductChange() {
    const orderType = document.querySelector('input[name="order-type"]:checked').value;
    
    if (orderType === 'kit') {
        // 零件包订单：使用零件包售价
        price = parseFloat(selectedOption.dataset.price);
    } else {
        // 整体产品订单：计算 BOM 总成本 × 2
        const bomData = await fetch(`${API_BASE}/products/${productId}/bom`);
        const totalCost = bomData.reduce((sum, item) => sum + item.total_cost, 0);
        price = totalCost * 2; // 零售价 = 成本 × 2
    }
}
```

**示例**:
- AP2401-SKU1 小推车
  - 零件包售价：¥221.76
  - 整体 BOM 成本：¥320.50
  - 整体零售价：¥641.00（成本 × 2）

---

### 问题 2: 零部件采购数量不对

**原因**: 直接使用 BOM 中的 `quantity`，没有乘以订单数量  
**修复**: 采购数量 = BOM 单个用量 × 订单数量

```javascript
const orderQuantity = order.quantity || 1;
const partsList = bomItems.map(item => ({
    // ...
    quantity: Math.ceil(item.quantity * orderQuantity), // 零件数量 × 订单数量
}));
```

**示例**:
- 订单：2 件小推车
- BOM: 4040 型材 × 8（单个用量）
- **修复前**: 采购 8 个 ❌
- **修复后**: 采购 16 个 ✅（8 × 2）

## 📝 注意事项

### 价格计算
- **零件包订单**: 使用 `dealer_price`（零件包售价）
- **整体产品订单**: BOM 总成本 × 2（零售价）
- **折扣功能**: 在计算后的价格基础上应用折扣

### 采购数量
- **单个用量**: BOM 中定义的单个产品零件数量
- **采购数量**: 单个用量 × 订单数量
- **向上取整**: 使用 `Math.ceil()` 确保数量充足

### 采购清单
- **过滤无链接**: 自动过滤没有淘宝链接的零件
- **可滚动表格**: 最多显示 350px 高度，超出可滚动
- **确认后才打开**: 用户必须点击"确认采购"才会打开淘宝
