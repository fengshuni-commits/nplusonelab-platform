## 📊 数据库结构

### users 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| username | TEXT | 用户名（唯一） |
| password_hash | TEXT | SHA256 密码哈希 |
| role | TEXT | 角色（admin/user） |
| status | TEXT | 状态（active/inactive） |
| created_at | TIMESTAMP | 创建时间 |
| last_login | TIMESTAMP | 最后登录时间 |
| purchase_count | INTEGER | 采购次数 |

### user_logs 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| user_id | INTEGER | 用户 ID（外键） |
| action | TEXT | 操作类型 |
| detail | TEXT | 操作详情 |
| created_at | TIMESTAMP | 创建时间 |

## ⚠️ 注意事项

### 安全性
1. ⚠️ 密码使用 SHA256 哈希（基础加密）
2. ⚠️ 建议升级为 bcrypt 或 argon2
3. ⚠️ 需要添加 JWT 或 session 认证
4. ⚠️ 需要添加密码重置功能

## 🔧 故障排查

### 问题 1: API 返回 Internal Server Error
**解决**:
```bash
# 检查后端日志
tail -f /tmp/n1lab_backend.log

# 重启后端服务
pkill -f uvicorn
cd /root/.openclaw/workspace/projects/n1-lab-automation/backend
uvicorn main:app --host 0.0.0.0 --port 8000
```

### 问题 2: 数据库表不存在
**解决**:
```bash
cd /root/.openclaw/workspace/projects/n1-lab-automation/backend
python3 init_users.py
```

### 问题 3: 创建用户失败
**检查**:
- 用户名是否已存在
- 用户名格式是否正确（3-20 位字母数字下划线）
- 密码长度是否≥6 位
